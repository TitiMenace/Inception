-faire une petite infrastructure avec des services sous certaines regles.
-Je dois utiliser Docker Compose
-Chaque image docker doit avoir le meme nom que son service correspondant
-Chaque service doit run dans son container correspondant.
-Chaque container doit run soit la penultieme version de debian ou alpine.
-On doit ecrire nos propres dockerfile, un par service. Les Dockerfiles doivent etre appeles
dans mon docker-compose.yml qui est appele par le Makefile, ce qui signifie que on doit build nos propores
docker images

Setup :

-  Un docker container qui contient NGINX avec TLSv1.2 ou TLSv1.3 seulement
-	Un docker container qui contient WordPress avec php-fpm (doit etre installe et configurer), sans nginx.
- Un docker container qui contient MariaDB, sans nginx.
- Un volume qui contient la databate de mon Wordpress
-	un volume qui contient mes fichiers de mon site Wordpress
- Un docker-network qui etablit la connection entre mes containers

Si ca crash mon mes containers doivent crash.

-Regarder comment les Daemons fonctionnent
-La ligne network ? doit etre presente dans mon docker-compose.yml.
-Les containers ne doivent pas etre run avec une commande qui lance une boucle inf
- pas de : -tail f, bash, sleep infinity, while true.
- regarder le PID1, et les meilleurs methodes pour ecrire les dockerfiles

-	Dans la database du WP, il doit y avoir 2 users, parmis eux 1 admin.
-	l'username  de l'admin ne doit pas contenir admin ou un truc qui y fait reference ou y ressemble.
-	Les volumes doivent etre accessibles dans le /home/login/data dossier de la host machine qui fait tourner docker. On
remplace evidemment le login par le mien.

-Pour facilite le processus, je doit configurer le nom de domaine pour qu'il pointe vers mon adresse IP locale.
-Ce nom de domaine doit etre login.42.fr. Le nom de domaine doit redirige vers mon adresse IP qui redirige vers mon site.

- Latest tag ? est interdit
- Il ne doit pas y avoir de mot de passe dans un dockerfile
- l'utilisation de variable d'env ? est mandatory
- C'est fortement recommande d'utiliser un .env pour stocker les var d'env et d'utiliser le Docker secrets ? 
pour stocker des informations confidentielles.

en utilisant les protocoles TLSv.1.2 ou TLSv.1.3
